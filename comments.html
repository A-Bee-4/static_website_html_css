Webpages structure

Across the website there are different usages of semantic HTML that have structured our website, not just for accessibility in mind but for better SEO and overall experience.

Header

The header contains the website title "Web Systems". The website title is written using the h1 tag, and its properties are defined by the header class in the CSS file. Each pages have a header tag which wraps our logo and navigation, the navigation's links are link tags, which redirect us to different pages.

Main

The main tag is what wraps our content of the page and this contains all sections to divs into the span of our contents. Some images are also contained within their own divs to be moved other positions using CSS.

Footer

As the name suggest footer is used to display footer related contents - store extra information which users will always be able to find them.

Aesthetics

The wbsites aethstetics are designed to be appealing, yet clear and easy to navigate and read. Here are some points outlining some of the aethstetical features of the website.

  • Page heading are bold to avoid confusion and a horizontal rule is used to make the content heading stand out from the actual page information.
  • Padding has been used inside the main content container, between images and text, and inside tables to page the website look less cluttered.
  • Empty space on the sides, space on both top and bottom of the page have been added to make the page look more presentable and clean.

Website Performances / Accessibility / Color Contrast review

While conducting and auditing the website we've put far more interest into accessibility, SEO and experience of the site. we're surprise by the result we've got as provide in the image. Images speaks 1000 words so no need to explain when scores are 100!

Fonts

We've made the use of google fonts Open Sans for better readability and user experience. Furthermore we've a fallback font which is sans-serif if the first font is not loaded the site will fallback to using the default, Aside from using google fonts we've used variable font sizing, fairly new CSS function clamp which allows us to have variable font sizing, This ensures users will have better reading experience.

Animations / Interactivity

The nav links uses micro animation which indicates which pages user is on and helps users while hovering on them, this helps and engages user to explore our site more.

Structure

The structure of the website has been laid out vertically. This provides the user with a cleaner layout than with a navigation sidebar or using a two column layout, and makes the content in front of them stand out better. It also makes viewing the website on a mobile device much easier due to the extra horizontal screen real estate.

Accessibility

This website has been designed to be assessible to page users as possible. Here are some points outlining some of the features that make this website more readily accessible.

  • The pages are W3C validated, with no errors or any warnings. W3C validation has many enduring advantages, such as extending the lifespan of the website pages, maintaining browser compatibility, search engine performance, device compliance and much more.
  • the "alt" attribute has been set on all images to give users with visual impairments a general idea of what the image is about without actually having to see the image.
  • The text used on the web page contrasts highly with the background colour, which also makes it easier for user to read the information on the page. The page title is large and the menu items are clear and succinct on the page.
  • Relative font sizes are used for all textual elements of the page, so that the users may freely resize the text on their browser using their cleints zoom controls.

User Interfaces / Responsive design

Throughout our site we've used many techniques to ensure consistency, we've made the use of CSS variable to store all colors used throughout the website, this helps with consistency of design and reduce amount of duplicate code.

Spacing

To keep the spacing in this design consistent and boost our development, we've created pre defined classes spacing, this not only helps us in early stage of designing but long term this reduces code duplication.

Responsiveness

We've kept the website structure simple and adjusted website in mobile to make it responsive for better user experience.

Colour Contrast

Particular attention has been paid to colour contrast when designing the aesthetics of the website. Colour combinations that do not strain the eye are used throughout the website. The website has been tested for color contrast using Coolors , with successful results.
